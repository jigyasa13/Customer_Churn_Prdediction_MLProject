# Customer Churn Prediction

Hey there! Welcome to my repository. In this project, I built an end-to-end Machine Learning pipeline to analyze customer behavior data and predict whether a customer is likely to leave or stay with a service (churn). 

Predicting churn is incredibly valuable because it helps companies identify at-risk users early and implement proactive customer retention strategies to keep them happy.

## 🚀 Project Overview
This project uses past customer behavior metrics—including demographics, account information, and usage patterns—to classify if a customer will churn. I handled the data exploration, structural preprocessing, and predictive model training to establish a reliable baseline classification.

## 🛠️ What's Inside the Code?
Here is the general structure of the workflow implemented in the Jupyter Notebook:
1. Data Gathering & Exploration: Loaded the customer database and analyzed the target variable balance alongside missing data attributes.
2. Data Preprocessing & Cleaning: Structured categorical column modifications, handled missing inputs safely, and mapped label constraints into binary formats.
3. Feature Engineering & Scaling: Applied robust encoders to match nominal/ordinal criteria and scaled features to prevent model bias.
4. Classification Model Training: Standardized data subsets, split into training and testing components, and ran optimization models to achieve high evaluation scores.
